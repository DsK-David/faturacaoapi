Introdução

A Faturação API é uma API que facilita o gerenciamento de vendas, clientes, produtos e entidades. Ela permite criar faturas automáticas em PDF e fornece uma interface para interagir com os dados de forma segura e eficiente.

Endpoints

Clientes

  • GET /api/v1/cliente - Retorna todos os clientes cadastrados.
  • GET /api/v1/cliente/:entidadeID - Retorna todos os clientes de uma entidade específica.
  • POST /api/v1/cliente/ - Envia os dados do cliente para o banco de dados (DESIG,EMAIL,TELEFONE,Entidade_ID)
  • DELETE /api/v1/cliente/:clienteID/:entidadeID - Deleta um cliente específico.
    Parâmetros:
    - :clienteID: ID do cliente.
    - :entidadeID: ID da entidade a qual o cliente pertence.

Vendas

  • GET /api/v1/vendas - Retorna todas as vendas realizadas.
  • GET /api/v1/vendas/:id - Retorna todas as vendas associadas a uma entidade específica.
    Parâmetro:
    - :id: ID da entidade.
  • POST /api/v1/venda - Adiciona uma nova venda e gera uma fatura em PDF.

Produtos

  • GET /api/v1/produto - Retorna todos os produtos cadastrados.
  • GET /api/v1/produto/:entidadeID - Retorna produtos de uma entidade específica.
    Parâmetro:
    - :entidadeID: ID da entidade.
  • GET /api/v1/produto/barcode/:barcode - Retorna um produto pelo código de barras.
    Parâmetro:
    - :barcode: Código de barras do produto.
  • GET /api/v1/produto/nome/:produto_nome/:entidadeID - Retorna um produto pelo nome.
    Parâmetros:
    - :produto_nome: Nome do produto.
    - :entidadeID: ID da entidade a qual o produto pertence.
  • GET /api/v1/produto/categoria/:categoriaID - Retorna produtos de uma categoria específica.
    Parâmetro:
    - :categoriaID: ID da categoria.

Categorias

  • GET /api/v1/categoria - Retorna todas as categorias cadastradas.
  • GET /api/v1/categoria/:id - Retorna uma categoria específica.
    Parâmetro:
    - :id: ID da categoria.

Autenticação

  • GET /api/v1/auth/:username/:password - Autentica um usuário com seu nome de usuário e senha.
    Parâmetros:
    - :username: Nome de usuário.
    - :password: Senha do usuário.

Entidades

  • GET /api/v1/entidade/:id - Retorna informações de uma entidade específica.
    Parâmetro:
    - :id: ID da entidade.

Autenticação

A API usa chaves de API para autenticar solicitações. Para acessar os endpoints protegidos, você deve incluir um cabeçalho Authorization com a sua chave de API em todas as solicitações.

Exemplos de Uso


// Exemplo de requisição GET para listar todos os clientes
fetch('/api/v1/cliente', {
    method: 'GET',
    headers: {
        'Authorization': 'SuaApiKeyAqui'
    }
})
.then(response => response.json())
.then(data => console.log(data));

Tratamento de Erros

A API retorna códigos de status HTTP para indicar o sucesso ou falha das solicitações. Os códigos mais comuns incluem:

  • 200 OK: A solicitação foi bem-sucedida.
  • 400 Bad Request: A solicitação foi malformada ou contém dados inválidos.
  • 401 Unauthorized: A chave da API está ausente ou é inválida.
  • 404 Not Found: O recurso solicitado não foi encontrado.
  • 500 Internal Server Error: Ocorreu um erro no servidor.
